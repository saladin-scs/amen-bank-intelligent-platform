# Amen Bank Knowledge Base Dataset

Enterprise knowledge base for the **Amen Bank Intelligent Digital Banking Platform** AI Engine.

This dataset corresponds to the **CRISP-DM Data Preparation** phase. It is designed for embedding generation, ChromaDB indexing, semantic retrieval, RAG chat, product recommendation, and multilingual Q&A.

## Privacy rules (mandatory)

This knowledge base stores **bank service knowledge only**.

Do **not** store:

- account numbers
- balances or monetary demo values tied to customers
- transaction amounts / personal operation dates
- login credentials
- any client-specific information

Demo digital banking screens may exist publicly; they must never be ingested as factual customer data.

## Cleaning process

Implemented in `ai-engine/preprocessing/`:

1. **`clean_documents.py`**
   - BeautifulSoup HTML stripping
   - removal of navigation / footer artifacts
   - unicode / whitespace normalization
   - privacy regex guards (IBAN-like, card-like, balance dumps, credentials)
   - fingerprint-based deduplication
2. **`normalize_text.py`**
   - builds a canonical retrieval text from title + narrative + structured facets (workflows, eligibility, FAQs, etc.)
   - normalizes language codes and metadata
3. **`chunk_documents.py`**
   - ≈ **500 tokens** / chunk
   - **100 tokens** overlap
   - sentence-aware splitting with metadata on every chunk
4. **`generate_metadata.py`**
   - source catalog, category/language stats, privacy attestation
5. **`run_pipeline.py`**
   - end-to-end orchestrator writing processed artifacts

### Run the pipeline

```bash
cd ai-engine/preprocessing
pip install -r requirements.txt
python run_pipeline.py
```

Outputs:

- `datasets/processed/documents.json`
- `datasets/processed/chunks.json`
- `datasets/metadata/sources.json`
- `datasets/metadata/quality_report.csv`

## Preparation for embeddings & ChromaDB

Ready pipeline:

```text
Raw Data → Cleaning → Structured Documents → Chunking → Embeddings → ChromaDB → RAG Assistant
```

Suggested next steps (AI Engine phases after this dataset):

1. Choose a multilingual sentence-transformer encoder
2. Embed each chunk from `chunks.json`
3. Persist vectors + metadata in ChromaDB (`ai-engine/vector_database/`)
4. Implement manual retrieval (top-k cosine / ANN) before LLM generation
5. Evaluate with `test_questions.json`
